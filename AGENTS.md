# AGENTS.md

Guidance for AI agents working in this repo (`opik.dev`, the personal website of Taufik Bahri / "Opik").

## What this is

A static personal website + blog. Astro 5 (static output) + Tailwind CSS v4, content authored in Markdown and edited via Decap CMS. Deployed as static files (Netlify — uses Git Gateway for the CMS).

## Commands

Package manager is **pnpm** (do not use npm/yarn).

| Command | Action |
| :-- | :-- |
| `pnpm install` | Install dependencies |
| `pnpm dev` | Dev server at `localhost:4321` |
| `pnpm build` | Production build to `./dist/` |
| `pnpm preview` | Preview the build locally |
| `pnpm astro check` | Type-check (`.astro` + TS strict) |

There is no test suite, linter, or formatter configured. Validate changes with `pnpm astro check` and `pnpm build`.

## Stack

- **Astro 5**, static output (no SSR adapter). Zero JS by default; interactivity is opt-in per island.
- **Tailwind CSS v4** via the `@tailwindcss/vite` plugin — there is **no `tailwind.config`**. Theme tokens and custom utilities live in `src/styles/global.css` under `@theme` / `@layer`.
- **TypeScript** strict (`astro/tsconfigs/strict`), 2-space tab indentation.
- **Inter** font, self-hosted via `@fontsource/inter` (imported in `global.css`).
- **Decap CMS** at `/admin` (Git-based, `git-gateway`, `main` branch) — config in `public/admin/config.yml`.
- **@astrojs/sitemap** auto-generates `sitemap-index.xml`; `robots.txt` is generated by `src/pages/robots.txt.ts`.

## Layout

- `src/pages/` — routes: `index`, `about`, `contact`, `404`, `blog/index` (listing), `blog/[...slug]` (post detail).
- `src/layouts/Layout.astro` — the single layout; wraps `BaseHead`, `Header`, `Footer`, `ConsentBanner`.
- `src/components/` — `BaseHead` (SEO/OG/JSON-LD), `Header`, `Footer`, `ConsentBanner`.
- `src/content/blog/*.md` — blog posts. Collection + frontmatter schema in `src/content/config.ts`.
- `src/consts/site.ts` — **single source of truth** for site title, URL, author, social links, email, GTM ID.
- `src/styles/global.css` — theme tokens, `.container-compact`, `.markdown` prose styles, `.sr-only`.
- `public/` — static assets, `public/images/blog/` (CMS media folder), `public/admin/` (Decap CMS).

## Conventions

- **Dark theme only.** `<html class="dark">` is hardcoded; there is no light mode. Use the **zinc** palette exclusively, with `zinc-50` (near-white) as the accent. Don't introduce other colors.
- **Site metadata** (author name, URLs, socials, email) always comes from `src/consts/site.ts` — never hardcode these in pages/components.
- **Container:** wrap page content width with `.container-compact` (`max-w-2xl mx-auto px-6`).
- **Buttons & tags:** pill-shaped (`rounded-full`).
- **Accessibility is a standard, not optional.** Every interactive element has `focus-visible:ring-*`, appropriate `aria-*`/`role`, and the layout includes a skip-link + live regions. Match this bar in new UI.
- **SEO is thorough.** New page types should add the appropriate JSON-LD (`schema.org`) block, matching the existing pattern (`Person`, `WebSite`, `Blog`, `BlogPosting`, `BreadcrumbList`, `AboutPage`, `ContactPage`). `BaseHead` handles OG/Twitter meta via props (`type`, `publishedTime`, `tags`, `image`, etc.).
- **Analytics is privacy-first.** GTM loads only after explicit consent (`ConsentBanner`) and only if `PUBLIC_GTM_ID` is set. Currently unset, so the banner is hidden by default. Don't add tracking that bypasses consent.

## Blog authoring

**Before writing or editing post content, read [CONTENT_STYLE.md](CONTENT_STYLE.md)** — it defines the required writing format, structure, and tone for posts (short quotable paragraphs, bolded lead-ins, smooth ordered flow, a mandatory Bottom Line / Key Takeaways summary, and a conversational-but-sharp tone). This section below only covers the mechanical/frontmatter side.

Posts are Markdown in `src/content/blog/` with YAML frontmatter (schema in `src/content/config.ts`):

```yaml
---
title: '...'              # required
description: '...'        # required
pubDate: 'YYYY-MM-DD'     # required
updatedDate: 'YYYY-MM-DD' # optional
heroImage: '/images/blog/<name>.webp'  # optional; store in public/images/blog/
tags: ['web-development', 'astro']     # optional, defaults to []
---
```

- Hero images live in `public/images/blog/` (the Decap CMS media folder) — prefer `.webp`.
- Post body is styled by the `.markdown` utility in `global.css` (the post template wraps `<Content />` in `<div class="markdown">`).
- The blog index (`blog/index.astro`) does **client-side** search / tag-filter / pagination (10/page) by serializing post data into a `define:vars` script.

## Known gotchas

- **Reading-time logic is duplicated** in `index.astro`, `blog/index.astro`, and `blog/[...slug].astro` (200 wpm). If you change it, change all three (or extract a shared util).
- **`?tag=` deep links don't pre-filter.** Post pages and the home page link to `/blog?tag=foo`, but `blog/index.astro` reads tags from button clicks and ignores `location.search`. A `?tag=` URL lands on the unfiltered list.
- `README.md` is still the default Astro starter boilerplate (not project-specific).
